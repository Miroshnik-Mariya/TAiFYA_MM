TYPE <список описаний типов>
<список описаний типов>::=<описание>|<описание>,<список описаний типов>
<описание>::=<идентификатор> = <простой тип>|<тип массив>|<тип указатель>
<простой тип>::= CARDINAL | INTEGER | REAL | CHAR | BOOLEAN
<тип массив> :: = ARRAY <диапазоны> OF <простой тип>
<диапазоны> :: = <диапазон>|<диапазоны>,<диапазон>
<диапазон> :: = [ <константа1> .. <константа2> ]
<тип указатель> :: = POINTER TO <простой тип>
<идентификатор> - последовательность символов языка начинается с буквы, включает
последовательность букв, цифр
<константа1>, <константа2> - целые числа без знака

Семантика:
Идентификатор имеет длину не более 8 символов и не совпадает с
зарезервированными словами TYPE, CARDINAL,| INTEGER, REAL, CHAR, BOOLEAN,
ARRAY, OF, TO, POINTER. Недопустимо дублирование имен при определении их типов.
Константа1, константа2 находятся в диапазоне 1 .. 256 и константа1 &lt; константы2
Вывести неповторяющиеся списки идентификаторов, с указанием их типа и объема
памяти, при этом CHAR занимает 1 байт; REAL – 8 байт; INTEGER- 4 байта,
CARDINAL - 2 байта.
Сообщать об ошибках при анализе цепочек языка, указывая курсором место
ошибки и ее содержание.
Анализатор работает до первой ошибки, допускает произвольное число пробелов
между конструкциями оператора (пробелы могут и отсутствовать), не учитывается
регистр символов.

Примеры правильных цепочек:
TYPE GF = ARRAY [ 7 .. 78 ], [ 98 .. 255 ] OF CHAR, G = POINTER TO BOOLEAN;
TYPE G45F = Cardinal, G2 = POINTER TO char, t54 = Array [2..22] of Integer;
TYPE G45F = Cardinal, G2 = POINTER TO char, t54 = Array [2..22] of Integer, G435F = Cardinal, G32 = POINTER TO char, t534 = Array [2..22] of Integer, G485F = Cardinal, G82 = POINTER TO char, t584 = Array [2..22] of Integer;
